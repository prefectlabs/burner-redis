**Discovery (forced the `cargo-bundle-licenses`-in-`build.script` pattern):**
`THIRDPARTY.yml` was committed to the repo in `6f017ce` (Plan 13-02, 2026-04-18),
AFTER the `v0.1.2` tag was cut. The PyPI `burner_redis-0.1.2.tar.gz` sdist does
not contain `THIRDPARTY.yml` at its root. Therefore the recipe cannot reference
`THIRDPARTY.yml` in `about.license_file` unless the build step generates it
first. The pydantic-core pattern (`cargo-bundle-licenses --format yaml --output
THIRDPARTY.yml` as the second command in `build.script`) is exactly the right
mitigation, so we mirror it. This also means conda-forge CI will re-run the
license audit on every build — the same pre-flight we did in Plan 13-02 will
run upstream.

Verbatim contents:

```yaml
schema_version: 1

context:
  name: burner-redis
  version: "0.1.2"

package:
  name: ${{ name|lower }}
  version: ${{ version }}

source:
  url: https://pypi.org/packages/source/b/burner-redis/burner_redis-${{ version }}.tar.gz
  sha256: 189698190835809f73fdb5af9ead4962975181c7fc8297045a5d831c0d465add

build:
  number: 0
  skip: target_platform == "linux-ppc64le"
  script:
    - python -m pip install . -vv
    - cargo-bundle-licenses --format yaml --output THIRDPARTY.yml

requirements:
  build:
    - if: build_platform != target_platform
      then:
        - python
        - cross-python_${{ target_platform }}
        - maturin >=1,<2
    - ${{ compiler('c') }}
    - ${{ stdlib('c') }}
    - ${{ compiler('rust') }}
    - cargo-bundle-licenses
  host:
    - pip
    - python
    - maturin >=1,<2
  run:
    - python

tests:
  - python:
      imports:
        - burner_redis
      pip_check: true
  - requirements:
      run:
        - pip
    script:
      - pip check
      - python -c "import burner_redis; burner_redis.BurnerRedis()"

about:
  summary: An embedded, in-process Redis-compatible database
  license: MIT
  license_file:
    - LICENSE
    - THIRDPARTY.yml
  homepage: https://github.com/PrefectHQ/burner-redis
  repository: https://github.com/PrefectHQ/burner-redis

extra:
  recipe-maintainers:
    - desertaxle
```

**2. [Rule 1 — Bug] `about.license_file: THIRDPARTY.yml` requires the file to exist in the build workdir; 0.1.2 sdist doesn't ship it — mirrored pydantic-core's `build.script` pattern instead of the plan's one-line script**

- **Found during:** Task 1 Step 1.3 (draft), while cross-checking the plan's
  interfaces template vs. pydantic-core's current template.
- **Issue:** The plan's interfaces block specifies
  `about.license_file: [LICENSE, THIRDPARTY.yml]` AND a single-line
  `build.script: python -m pip install . -vv --no-build-isolation`. But
  `THIRDPARTY.yml` was committed in `6f017ce` — AFTER the `v0.1.2` PyPI release.
  The 0.1.2 sdist therefore does not contain `THIRDPARTY.yml` at its root, so
  `rattler-build` would fail `about.license_file` validation.
- **Fix:** Followed pydantic-core-feedstock verbatim: expand `build.script` to
  a list that first runs `pip install . -vv` (drops the
  `--no-build-isolation` flag from the plan — pydantic-core doesn't use it,
  and `maturin` coming from `requirements.host` means pip's default isolation
  is fine) and then runs
  `cargo-bundle-licenses --format yaml --output THIRDPARTY.yml`. This
  guarantees THIRDPARTY.yml exists in the build workdir before
  `about.license_file` is resolved.
- **Files modified:** `/tmp/phase-13-staged-recipes/recipe-draft.yaml`.
- **Verification:** `conda-smithy recipe-lint --conda-forge` → clean. Pattern
  matches pydantic-core, which is the plan-mandated primary template.
- **Downstream impact:** None for the submission. The recipe still embeds
  third-party license text in `about.license_file` per conda-forge policy; the
  only change is WHEN the file is produced (build time, not sdist time).

### Informational (not a deviation)

- The plan's interfaces block suggested `build.skip: linux and ppc64le` as one
  candidate syntax. V1 `rattler-build` uses boolean expressions, not v0
  selectors. The example-v1 recipe and pydantic-core both use
  `target_platform == "linux-ppc64le"`-style comparisons. Chose
  `skip: target_platform == "linux-ppc64le"` as the single-string form that
  matches current v1 idiom. conda-smithy accepted it with no warning.
