  <action>
Add a public method `sweep_expired(&self)` to the `Store` impl block. This method:

1. Acquires a READ lock on `self.data`.
2. Collects all keys that have `expires_at = Some(...)` into a Vec (these are candidates).
3. Drops the read lock.
4. If there are no candidates, return immediately.
5. Randomly sample up to 20 keys from the candidates. Use a simple deterministic shuffle: iterate candidates and pick every Nth element to get ~20, or if fewer than 20 total, use all of them. For randomness, use `std::collections::hash_map::DefaultHasher` to hash `Instant::now()` as a seed and shuffle indices. Alternatively, a simpler approach: just take the first 20 candidates (since HashMap iteration order is already pseudo-random due to hashing). Use the simpler approach -- HashMap iteration is effectively random.
6. For each sampled key, acquire a WRITE lock, check if the key still exists and is expired via `is_expired()`, and if so remove it. Release the write lock after each check-and-remove cycle (or batch them in a single write lock acquisition for efficiency).

Efficient implementation: Acquire a single write lock, iterate all entries, collect expired keys (up to 20 with TTLs), and remove them in one pass. This avoids multiple lock acquisitions.

```rust
/// Sweep up to 20 expired keys from the keyspace.
/// Called periodically by the background expiration task.
/// Returns the number of keys removed.
pub fn sweep_expired(&self) -> usize {
    let mut data = self.data.write();
    let mut to_remove = Vec::new();
    let mut checked = 0;

    for (key, entry) in data.iter() {
        if entry.expires_at.is_some() {
            checked += 1;
            if entry.is_expired() {
                to_remove.push(key.clone());
            }
            if checked >= 20 {
                break;
            }
        }
    }

    let count = to_remove.len();
    for key in to_remove {
        data.remove(&key);
    }
    count
}
```

fn test_sweep_expired() {
    let store = Store::new();
    // Set 3 keys: 2 with very short TTL (already expired by the time we sweep), 1 with no TTL
    store.set(
        Bytes::from("exp1"), Bytes::from("v1"),
        Some(Duration::from_millis(1)), false, false,

    let removed = store.sweep_expired();

    // Expired keys are gone
    assert_eq!(store.get(&Bytes::from("exp1")), None);

3. Use `pyo3_async_runtimes::tokio::get_runtime().spawn(async move { ... })` to spawn the sweep loop.
4. The spawned task:
   - Uses `tokio::time::interval(Duration::from_millis(100))`.
   - In a loop: `interval.tick().await`, then try to upgrade the `Weak<Store>`.
   - If upgrade succeeds (`Some(store)`), call `store.sweep_expired()`.
   - If upgrade fails (`None`), all `Arc<Store>` references are dropped, so break the loop (task exits).

Updated `new()` method:

```rust
use std::sync::Weak;
// ... (keep existing imports)

#[new]
fn new() -> Self {
    let store = Arc::new(Store::new());

    // Spawn background sweep task for active expiration (EXP-03).
    // Uses Weak<Store> so the task stops when all BurnerRedis instances are dropped.
    let weak_store = Arc::downgrade(&store);
    pyo3_async_runtimes::tokio::get_runtime().spawn(async move {
        let mut interval = tokio::time::interval(std::time::Duration::from_millis(100));
        loop {
            interval.tick().await;
            match weak_store.upgrade() {
                Some(store) => {
                    store.sweep_expired();
                }
                None => break, // Store dropped, stop sweeping
            }
        }
    });

    BurnerRedis { store }
}
```
