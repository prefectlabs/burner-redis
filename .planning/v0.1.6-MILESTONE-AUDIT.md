## Issues (Non-Critical)

### ISSUE-1: NoScriptError class never raised
- **Severity:** minor
- **Affected:** D-08 (Phase 12)
- **Location:** `src/lib.rs:1860`, `python/burner_redis/__init__.py:26-38`, `tests/test_scripting.py:85`
- **Description:** Class is defined and exported in `__all__`, but `evalsha` raises `ResponseError` via `make_response_error()`. Callers using `except NoScriptError` will silently fail to catch the error. Test masks the type mismatch via `pytest.raises(Exception, match="NOSCRIPT")`.
- **Remediation:** Add `make_noscript_error()` in lib.rs; route the `NOSCRIPT` error message through it; tighten the test.

### ISSUE-2: Pipeline runtime error for zrangestore/zcount
- **Severity:** minor
- **Affected:** ZSET-04, ZSET-06
- **Location:** `python/burner_redis/pipeline.py:195-201`, `src/lib.rs:3823-3824`
- **Description:** Pipeline stubs queue commands that fall through `dispatch_pipeline_command` to the unknown-command arm at runtime.
- **Remediation:** Add `"zrangestore"` and `"zcount"` arms to `dispatch_pipeline_command` mirroring the standalone pymethods.
- **Status (Phase 15):** Already wired in commit de9d259 (2026-04-15) — Phase 15 added regression coverage in tests/test_pipeline.py.

### ISSUE-3: List persistence has no round-trip test
- **Severity:** minor (coverage gap, not a wiring gap)
- **Affected:** PERS-01..04, LIST-01..16
- **Location:** `src/persistence.rs:84-236`, `tests/test_persistence.py`
- **Description:** Production code correctly serializes/deserializes `ValueData::List` via `PersistableValueData::List` (`src/store.rs:3476-3478`), but neither the Rust unit test nor the Python persistence suite includes a List-type round-trip case.
- **Remediation:** Add a Rust unit test in `test_round_trip_all_types` and a Python `test_list_persistence` covering save/load on a populated list.
