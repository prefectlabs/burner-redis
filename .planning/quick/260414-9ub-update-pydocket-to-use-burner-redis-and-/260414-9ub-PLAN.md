**Rust-side (src/store.rs + src/lib.rs) — new commands:**

2. **hexists(key, field)** — Return True/False if field exists in hash. Used by schedule Lua

   In Lua dispatch, add "HEXISTS" case returning 0 or 1.

3. **zcard(key)** — Return cardinality (member count) of a sorted set. Used by
   `Worker.check_for_work()`. Implementation: read lock, get sorted set, return members len.
   Return 0 for missing keys.

4. **expire(key, seconds)** — Set TTL on an existing key. Used by worker heartbeat, execution
   TTL management. Implementation: write lock, if key exists set its expiry to
   `Instant::now() + Duration::from_secs(seconds)`. Return True/1 if key exists, False/0 if not.
   Accept integer seconds parameter.

5. **xdel(stream, *ids)** — Delete specific entries from a stream by ID. Used by
   `Worker.ack_message()`. Implementation: write lock, get stream, remove entries matching
   the given IDs. Return count of entries actually deleted.

6. **xrange(stream, min, max, count=None)** — Return stream entries in ID range. Used by
   `Docket.snapshot()`. Implementation: read lock, get stream, filter entries where

  <action>
Create `tests/test_pydocket_compat.py` that monkey-patches pydocket's `RedisConnection` to use
BurnerRedis, then exercises the full Docket + Worker lifecycle.

**Monkey-patch strategy:**

The key insight: pydocket's `RedisConnection` class (in `docket._redis`) manages connections.
Its `client()` context manager yields a `redis.asyncio.Redis` instance. We need to:

1. Create a shared `BurnerRedis` instance per test
2. Monkey-patch `RedisConnection.__aenter__` to be a no-op (skip real connection setup)
3. Monkey-patch `RedisConnection.client()` to yield the shared BurnerRedis instance
4. Monkey-patch `RedisConnection.pubsub()` to yield `burner_redis.pubsub()`
5. Monkey-patch `RedisConnection.publish()` to call `burner_redis.publish()`
6. Set `RedisConnection.is_connected` property to return True
7. Monkey-patch `RedisConnection.__aexit__` to be a no-op

Use pytest `monkeypatch` fixture for clean patching that auto-reverts.

**Test structure:**

```python
import pytest
import asyncio
from unittest.mock import AsyncMock
from contextlib import asynccontextmanager
from burner_redis import BurnerRedis

pytestmark = pytest.mark.integration


@pytest.fixture
def burner():
    """Shared BurnerRedis instance for pydocket tests."""
    return BurnerRedis()


@pytest.fixture
def patch_pydocket(monkeypatch, burner):
    """Monkey-patch pydocket to use BurnerRedis instead of redis.asyncio.Redis."""
    from docket._redis import RedisConnection

    @asynccontextmanager
    async def fake_client(self):
        yield burner

    @asynccontextmanager
    async def fake_pubsub(self):
        ps = burner.pubsub()
        try:
            yield ps
        finally:
            await ps.aclose()

    async def fake_publish(self, channel, message):
        return await burner.publish(channel, message)

    async def fake_aenter(self):
        self._connection_pool = True  # Make is_connected return True
        return self

    async def fake_aexit(self, *args):
        self._connection_pool = None

    monkeypatch.setattr(RedisConnection, "client", fake_client)
    monkeypatch.setattr(RedisConnection, "pubsub", fake_pubsub)
    monkeypatch.setattr(RedisConnection, "publish", fake_publish)
    monkeypatch.setattr(RedisConnection, "__aenter__", fake_aenter)
    monkeypatch.setattr(RedisConnection, "__aexit__", fake_aexit)
```

1. **test_docket_add_immediate_task** — Create a Docket, add a task (AsyncMock), create a Worker,
   run `worker.run_until_finished()`, verify the mock was called. This exercises: xgroup_create,
   register_script (Lua scheduling), xadd, xreadgroup, xack, hset, hget, hgetall, pipeline,
   zadd, lock, publish.

2. **test_docket_add_delayed_task** — Schedule a task with `when=` in the near future (50ms from

   after the delay. This exercises: zadd (delayed queue), zrangebyscore (scheduler loop Lua),
   xadd (move to stream), the full delayed task lifecycle.

3. **test_docket_cancel_task** — Schedule a delayed task, cancel it before execution, verify it
   does not execute. This exercises: the cancel Lua script (ZREM + DEL + HSET).

4. **test_docket_snapshot** — Schedule multiple tasks, take a snapshot via `docket.snapshot()`,
   verify it reflects the scheduled tasks. This exercises: hgetall, zrange, xrange, pipeline
   with zcard.

5. **test_worker_heartbeat** — Start a worker, verify it registers heartbeat. This exercises:
   zadd, sadd, expire, pipeline, zremrangebyscore.
