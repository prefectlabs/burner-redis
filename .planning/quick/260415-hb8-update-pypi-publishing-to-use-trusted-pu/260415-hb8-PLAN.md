  <action>
In `.github/workflows/release.yml`, modify the `release` job:

1. Add the `environment` block immediately after `runs-on: ubuntu-latest`:
   ```yaml
   environment:
     name: pypi
     url: https://pypi.org/p/burner-redis
   ```
   This is required for PyPI trusted publisher matching -- the environment name in the workflow must match what is configured on PyPI's side.

2. Replace the "Publish to PyPI" step entirely. Remove:
   ```yaml
   - name: Publish to PyPI
     uses: PyO3/maturin-action@v1
     with:
       command: upload
       args: --non-interactive --skip-existing dist/*
     env:
       MATURIN_PYPI_TOKEN: ${{ secrets.PYPI_API_TOKEN }}
   ```
   Replace with:
   ```yaml
   - name: Publish to PyPI
     uses: pypa/gh-action-pypi-publish@release/v1
   ```
   No `with:` block needed -- the action reads from `dist/` by default (matching the download-artifact output) and attestations are enabled by default since v1.11.0. No `env:` block needed -- OIDC token exchange is automatic when `id-token: write` permission is set and no username/password is provided.

3. Do NOT add `skip-existing: true` -- tag-triggered releases always produce new versions, and failing on duplicate uploads is the correct safety behavior.

4. Keep all other steps (Download all artifacts, Create GitHub Release) unchanged.
5. Keep the existing `permissions:` block unchanged (`id-token: write` and `contents: write` are already correct).
