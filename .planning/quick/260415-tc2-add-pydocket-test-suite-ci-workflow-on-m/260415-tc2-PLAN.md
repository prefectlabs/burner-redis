Create a GitHub Actions workflow that checks out the pydocket repo (chrisguidry/docket) at its
`replace-fakeredis-with-burner-redis` branch, builds burner-redis from source, and runs
pydocket's test suite with `REDIS_VERSION=memory` to verify ongoing compatibility.

Purpose: Catch compatibility regressions between burner-redis and pydocket on every PR and merge to main.
Output: `.github/workflows/pydocket-compat.yml`
</objective>

  <action>
Create `.github/workflows/pydocket-compat.yml` with the following structure:

**Trigger:** Same as existing ci.yml — `push` to `main` and `pull_request` to `main`.

**Job: `pydocket-compat`** on `ubuntu-latest`:

1. **Checkout burner-redis** — `actions/checkout@v4` (default, checks out this repo)

2. **Set up Python 3.12** — `actions/setup-python@v5` with `python-version: "3.12"`

3. **Install Rust toolchain** — `dtolnay/rust-toolchain@stable`

4. **Install uv** — `astral-sh/setup-uv@v4`

5. **Create virtualenv** — `uv venv .venv`

6. **Build burner-redis from source** — activate venv, then `maturin develop --release`
   (this builds the current commit's Rust code and installs it into the venv)

7. **Checkout pydocket** — `actions/checkout@v4` with:
   - `repository: chrisguidry/docket`
   - `ref: replace-fakeredis-with-burner-redis`
   - `path: pydocket` (checkout into a subdirectory to avoid clobbering burner-redis)

8. **Install pydocket dependencies** — `uv pip install --python .venv/bin/python -e "./pydocket[metrics]" --group dev`
   This installs pydocket and its dev deps into the SAME venv that already has burner-redis.
   The `--python` flag targets the existing venv without needing `source .venv/bin/activate`.
   Include `[metrics]` optional dep since some tests may exercise metrics paths.

9. **Run pydocket tests** — Run from the `pydocket` directory:
   ```
   source .venv/bin/activate
   cd pydocket
   pytest tests/ -x --timeout=30 --numprocesses=logical --maxprocesses=4 -o "addopts="
   ```
   Key details:
   - Set env var `REDIS_VERSION: memory` (this makes pydocket use `memory://` URLs
     which route to BurnerRedis instead of spinning up Docker Redis containers)
   - Use `-o "addopts="` to OVERRIDE pydocket's default addopts from pyproject.toml
     (which includes --cov with 100% coverage requirement — we don't need/want coverage
     checks for this compatibility run, and the coverage paths won't match anyway)
   - Keep `--timeout=30` for safety
   - Keep `--numprocesses=logical --maxprocesses=4` for parallelism (pydocket uses pytest-xdist)
   - Use `-x` to fail fast on first error (this is a compatibility check, not a full report)
  </action>
