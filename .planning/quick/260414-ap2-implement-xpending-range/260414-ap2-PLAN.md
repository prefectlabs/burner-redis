**Store layer** (`src/store.rs`): Add `xpending_range` method after `xinfo_consumers`. Follow the same pattern as `xinfo_consumers` for stream/group lookup with passive expiration and NoGroup error handling. The method signature:

```rust
pub fn xpending_range(
    &self,
    key: &Bytes,
    group: &Bytes,
    min_id: StreamId,
    max_id: StreamId,
    count: usize,
    consumer_filter: Option<&Bytes>,
    min_idle_ms: Option<u64>,
) -> Result<Vec<(StreamId, Bytes, u128, u64)>, StoreError>
```

Returns Vec of (entry_id, consumer_name, idle_time_ms, delivery_count).

Implementation:
1. Acquire write lock for passive expiration (same pattern as xinfo_consumers)
2. Look up stream, return WrongType or NoGroup errors as appropriate
3. Iterate all consumers in the group (or just the filtered consumer)
4. For each consumer, iterate their `pending` HashMap entries
5. Filter entries where `entry_id >= min_id && entry_id <= max_id`
6. If `min_idle_ms` is Some, filter where `idle_time >= min_idle_ms`
7. Calculate idle_time_ms as `Instant::now().duration_since(pe.delivery_time).as_millis()`
8. Collect all matching entries into a Vec, sort by StreamId ascending
9. Truncate to `count` entries
10. Return the Vec of tuples

**PyO3 layer** (`src/lib.rs`): Add `xpending_range` method after `xinfo_consumers` in the `#[pymethods]` impl block. Use `#[pyo3(signature = (name, groupname, min="-", max="+", count=100, consumername=None, idle=None))]`. Note: redis-py passes min/max as str (e.g., "-", "+", "1234567890-0").

Parse min/max the same way as xrange ("-" -> (0,0), "+" -> (u64::MAX, u64::MAX), otherwise parse_stream_id).

Return format must match redis-py's `parse_xpending_range`: a Python list of dicts, each with keys:
- `b"message_id"` -> PyBytes of formatted stream ID
- `b"consumer"` -> PyBytes of consumer name
- `b"time_since_delivered"` -> int (milliseconds)
- `b"times_delivered"` -> int

Use `Python::try_attach` pattern consistent with xinfo_consumers/xautoclaim.

**Pipeline layer** (`python/burner_redis/pipeline.py`): Add `xpending_range` method after `xinfo_consumers`:
```python
def xpending_range(self, name, groupname, min="-", max="+", count=100, consumername=None, idle=None):
    self._commands.append(("xpending_range", (name, groupname, min, max, count), {"consumername": consumername, "idle": idle}))
    return self
```
